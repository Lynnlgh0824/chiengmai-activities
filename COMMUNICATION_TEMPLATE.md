# 需求确认模板

**使用场景**: 开发前快速对齐需求，避免返工
**预计节省时间**: 每个问题节省 10-30 分钟

---

## 📋 标准确认流程

### 第1步：快速诊断（必须回答）
```
1. 问题发生在什么时候？
   □ 页面加载时
   □ 用户操作后（具体操作：_____）
   □ 其他：_____

2. 当前行为：_____
3. 期望行为：_____
4. 截图/示例：有/无
```

### 第2步：数据范围确认
```
应该显示：
- Tab 0: ___ 个活动，规则：_____
- Tab 1: ___ 个活动，规则：_____
- Tab 2: ___ 个活动，规则：_____

明确排除：_____
```

### 第3步：实现确认
```
我理解的需求是：
- [ ] 当前：_____
- [ ] 期望：_____
- [ ] 触发条件：_____

对吗？（不对请指出）
```

## 🎯 常见场景速查

### 场景A：显示错误数据
```
□ 应该显示多少个？___ 个
□ 现在显示多少个？___ 个
□ 筛选规则是什么？（包含/排除）
□ 截图：有/无
```

### 场景B：Tab切换问题
```
□ 默认加载显示哪个Tab？ Tab ___
□ 点击后立即切换？是/否
□ 切换时清除筛选？是/否
□ 每个Tab的规则：_____
```

### 场景C：初始化问题
```
□ 页面首次加载状态：_____
□ 默认选中哪个Tab/选项？_____
□ 默认显示什么数据？_____
□ 默认筛选条件：_____
```

### 场景D：样式/布局
```
□ 期望的布局方式：_____
□ 参考截图/示例：_____
□ 移动端表现：_____
```

## 🔧 本次问题回顾

**问题描述**：页面默认加载时未按Tab筛选
**根本原因**：缺少明确的Tab初始化和筛选触发
**耗时**：约 20 分钟
**使用模板后预计**：3 分钟

**关键信号**：
- 🔴 "默认状态下"
- 🔴 "未点击tab时"
- 🔴 "整周的都是没有市集的数据"

**应该立即问**：
1. 默认加载时应该显示哪个Tab的数据？
2. 期望的筛选规则是什么？
3. 能给我看看当前显示的截图吗？

---

## 📝 使用说明

**Claude 使用时**：
1. 遇到需求不明确，立即打开此模板
2. 按照"快速诊断"提问
3. 等待用户回答，不要假设
4. 确认理解后，给出实现方案

**用户使用时**：
1. 按照"快速诊断"回答问题
2. 提供截图或示例
3. 用"当前vs期望"描述问题
4. Claude没理解时，换种说法
